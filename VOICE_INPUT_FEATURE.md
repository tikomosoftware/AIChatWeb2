# 音声入力機能の実装

## 概要

ワンダーランド東京 FAQ AIチャットボットに音声入力機能を追加しました。Web Speech APIを使用して、ユーザーがマイクから音声で質問を入力できるようになりました。

## 実装内容

### 1. カスタムフック: `useSpeechRecognition`

**ファイル:** `app/hooks/useSpeechRecognition.ts`

Web Speech APIをラップしたReactカスタムフックです。

**主な機能:**
- 音声認識の開始/停止
- リアルタイムでの音声テキスト変換
- エラーハンドリング
- ブラウザ対応チェック

**返り値:**
- `transcript`: 認識されたテキスト
- `isListening`: 音声認識中かどうか
- `isSupported`: ブラウザが対応しているか
- `error`: エラーメッセージ
- `startListening()`: 音声認識を開始
- `stopListening()`: 音声認識を停止
- `resetTranscript()`: テキストをリセット

### 2. MessageInputコンポーネントの拡張

**ファイル:** `app/components/MessageInput.tsx`

既存のメッセージ入力コンポーネントに音声入力機能を統合しました。

**追加機能:**
- マイクボタンの追加
- 音声認識中のビジュアルフィードバック
- 音声認識結果の自動入力
- 認識完了後の自動送信
- エラーメッセージの表示

## 使い方

1. チャット画面のメッセージ入力欄の横にあるマイクボタンをクリック
2. ブラウザがマイクへのアクセス許可を求めた場合は「許可」を選択
3. 音声で質問を話す
4. 話し終わると自動的に認識が完了し、メッセージが送信されます
5. 途中で停止したい場合は、もう一度マイクボタンをクリック

## 技術的な詳細

### Web Speech API

- **言語設定:** `ja-JP`（日本語）
- **認識モード:** `continuous: false`（単発認識）
- **中間結果:** `interimResults: true`（リアルタイム表示）

### セキュリティ要件

Web Speech APIは以下の環境でのみ動作します：
- HTTPS接続
- localhost（開発環境）

HTTPでは動作しないため、本番環境ではHTTPSが必須です。

### エラーハンドリング

以下のエラーケースに対応しています：

| エラーコード | 説明 | ユーザーへのメッセージ |
|------------|------|---------------------|
| `no-speech` | 音声が検出されない | 音声が検出されませんでした。もう一度お試しください。 |
| `audio-capture` | マイクにアクセスできない | マイクにアクセスできません。マイクの接続を確認してください。 |
| `not-allowed` | マイク使用が許可されていない | マイクの使用が許可されていません。ブラウザの設定を確認してください。 |
| `network` | ネットワークエラー | ネットワークエラーが発生しました。 |

## トラブルシューティング

### マイクが動作しない場合

1. ブラウザのマイク使用許可を確認
2. HTTPSまたはlocalhostでアクセスしているか確認
3. マイクが正しく接続されているか確認
4. 他のアプリケーションがマイクを使用していないか確認

### 音声が認識されない場合

1. 静かな環境で話す
2. マイクに近づいて、はっきりと話す
3. 日本語で話していることを確認
4. マイクの音量設定を確認
