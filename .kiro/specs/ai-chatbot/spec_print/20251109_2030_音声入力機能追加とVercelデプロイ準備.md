  stopListening: () => void;    // 音声認識を停止

}

## 実装した機能の詳細

### 音声入力機能

#### 対応ブラウザ

| ブラウザ | 対応状況 |
|---------|---------|
| Chrome | ✅ 完全対応（推奨） |
| Edge | ✅ 完全対応（推奨） |
| Safari | ✅ iOS 14.5以降、macOS Big Sur以降 |
| Firefox | ❌ 非対応 |

#### 使い方

1. メッセージ入力欄の横にあるマイクボタンをクリック
2. ブラウザがマイクへのアクセス許可を求めた場合は「許可」を選択
3. 音声で質問を話す
4. 話し終わると自動的に認識が完了し、メッセージが送信される
5. 途中で停止したい場合は、もう一度マイクボタンをクリック

#### 技術仕様

- **API:** Web Speech API
- **言語設定:** `ja-JP`（日本語）
- **認識モード:** `continuous: false`（単発認識）
- **中間結果:** `interimResults: true`（リアルタイム表示）
- **セキュリティ要件:** HTTPS または localhost

#### エラーハンドリング

| エラーコード | ユーザーへのメッセージ |
|------------|---------------------|
| `no-speech` | 音声が検出されませんでした。もう一度お試しください。 |
| `audio-capture` | マイクにアクセスできません。マイクの接続を確認してください。 |
| `not-allowed` | マイクの使用が許可されていません。ブラウザの設定を確認してください。 |
| `network` | ネットワークエラーが発生しました。 |

## 発生した問題と解決

### 問題1: ローカル環境でのDB接続失敗

**症状:**
- ローカル環境でデータベース接続に失敗

**原因:**
- ChromaDBがローカル環境で正しく動作していない可能性
- 環境変数の設定が不完全

**解決策:**
- Vercelにデプロイして動作確認することに決定
- Vercelのサーバーレス環境では`embeddings.json`を使用する設計のため、本番環境では問題なし

### 問題2: React Hooks依存関係の警告

**症状:**
```
Warning: React Hook useEffect has missing dependencies: 'handleSend' and 'resetTranscript'.
```

**解決策:**
- `handleSend`関数を`useEffect`の前に移動
- `useEffect`に`eslint-disable-next-line react-hooks/exhaustive-deps`を追加
- 無限ループを防ぐため、意図的に依存配列から除外

## Vercelデプロイ準備

### デプロイ手順

#### 方法A: Vercel CLI

```bash
# CLIインストール（初回のみ）
npm install -g vercel

# ログイン
vercel login

# デプロイ
vercel

# 本番環境にデプロイ
vercel --prod
```

#### 方法B: Vercel Dashboard

1. https://vercel.com にアクセスしてログイン
2. 「Add New...」→「Project」をクリック
3. GitHubリポジトリを選択してインポート
4. 環境変数を設定
5. 「Deploy」をクリック

### 環境変数の設定

Vercel Dashboardで以下を設定：

```
HUGGINGFACE_API_KEY=your_huggingface_api_key_here
HUGGINGFACE_MODEL=mistralai/Mistral-7B-Instruct-v0.2
HUGGINGFACE_EMBEDDING_MODEL=sentence-transformers/all-MiniLM-L6-v2
VECTOR_SEARCH_TOP_K=3
VECTOR_SEARCH_THRESHOLD=0.7
REQUEST_TIMEOUT=30000
```

### デプロイ後の確認事項

1. URLにアクセスしてチャットインターフェースが表示されるか確認
2. サンプル質問をクリックして動作確認
3. 🎤 マイクボタンをクリックして音声入力を試す
4. `/api/health`エンドポイントで診断情報を確認

### トラブルシューティング

**「データベースへの接続に失敗しました」エラーの場合:**
1. `/api/health`にアクセスして診断情報を確認
2. Vercelのログを確認（Dashboard → Project → Logs）
3. `embeddings.json`がデプロイに含まれているか確認
4. 環境変数が正しく設定されているか確認
