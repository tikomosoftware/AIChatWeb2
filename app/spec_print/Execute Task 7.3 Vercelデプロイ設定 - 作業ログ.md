### 4. デプロイ方針の変更

**ユーザー指示**:

  1. ChromaDB Cloudを使用
  2. 別のサーバーでChromaDBをホスト

**ユーザー質問**:

デメリット:
- スケーラビリティの制限（Vercelの50MB制限）
- パフォーマンスの低下（最適化されたインデックスが使えない）
- メンテナンス性（データ更新のたびに再デプロイ）
- メモリ使用量の増加

メリット:
- 追加のインフラ不要
- シンプルで管理が楽
- 小規模データなら十分実用的

## 学んだこと・注意点

1. **Vercelのサーバーレス制約**
   - ファイルベースのデータベースは動作しない
   - 各リクエストで新しいコンテナが起動する可能性がある

2. **データ埋め込み方式の適用範囲**
   - 小規模データ（数百件、数MB）なら最適
   - 大規模データは外部サービスが必要

3. **インターフェース設計の重要性**
   - 同じインターフェースを維持することで移行が容易
   - 将来の拡張性を考慮した設計

4. **ChromaDBのデータ構造**
   - `embeddings_queue`テーブルにデータが格納
   - ベクトルはBLOB形式で保存
   - メタデータはJSON文字列
