### 3. サブタスク 3.2: RAG処理フローの統合

**実装内容**:
RAGパイプライン全体を統合:

1. **クエリのベクトル化**:
   - `EmbeddingService.embedQuery()` を使用してユーザーメッセージをベクトル化
   - エラーハンドリング付き

2. **ベクトル検索の実行**:
   - `VectorSearchService.search()` でベクトルデータベースを検索
   - 類似度スコアによる自動フィルタリング

3. **類似度スコアのチェック**:
   - 検索結果が0件の場合、「関連情報が見つかりません」メッセージを返す
   - 要件に基づく閾値チェック（VectorSearchServiceで実装済み）

4. **LLMへのコンテキスト送信と回答生成**:
   - 検索結果からコンテキストドキュメントを抽出
   - `LLMService.generateResponse()` で回答を生成
   - ソース情報（スコア付き）を含めて返却

5. **タイムアウト処理**:
   - 環境変数 `REQUEST_TIMEOUT` から設定を読み込み（デフォルト30秒）
   - LLMサービスにタイムアウト値を渡す
   - タイムアウトエラーの適切なハンドリング

6. **エラーハンドリング**:
   - レート制限エラーの検出と処理
   - タイムアウトエラーの検出と処理
   - 各ステップでの詳細なエラーログ

**レスポンス形式**:
```typescript
interface ChatResponse {
  response: string;        // LLMが生成した回答
  sources?: string[];      // ソース情報（スコア付き）
  error?: string;          // エラーメッセージ（エラー時のみ）
}
```

**検証**:
- TypeScript診断でエラーがないことを確認
- タスクステータスを「完了」に更新

## 技術的な詳細

### エンドポイント仕様
- **URL**: `/api/chat`
- **メソッド**: POST
- **Content-Type**: application/json

### リクエスト形式
```json
{
  "message": "ユーザーの質問"
}
```

### レスポンス形式（成功時）
```json
{
  "response": "LLMが生成した回答",
  "sources": [
    "[1] Score: 0.85",
    "[2] Score: 0.78",
    "[3] Score: 0.72"
  ]
}
```

### レスポンス形式（エラー時）
```json
{
  "response": "",
  "error": "エラーメッセージ（日本語）"
}
```

### HTTPステータスコード
- `200`: 成功
- `400`: バリデーションエラー
- `500`: サーバーエラー
