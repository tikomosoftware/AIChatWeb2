## 実施内容

### 1. 環境変数ファイルへのSupabase設定追加

#### `.env.local` への追加
既存の環境変数ファイルに以下のSupabase設定を追加しました：

```env
# ============================================
# Supabase Configuration (オプション)
# ============================================
# Supabaseプロジェクト URL
# 取得方法: Supabaseダッシュボード > Settings > API
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url_here

# Supabase匿名キー（公開可能）
# 取得方法: Supabaseダッシュボード > Settings > API > anon public
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key_here
```

#### `.env.example` への追加
同様の設定を `.env.example` にも追加し、プレースホルダー値を設定しました：

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key_here
```

### 2. データベースセットアップSQLファイルの作成

**ファイル:** `.kiro/specs/chat-history-storage/supabase-setup.sql`

以下の内容を含むSQLファイルを作成しました：

#### テーブル定義
```sql
CREATE TABLE IF NOT EXISTS chat_history (
  id uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  question text NOT NULL,
  answer text NOT NULL,
  created_at timestamptz NOT NULL DEFAULT now()
);
```

#### インデックス作成
```sql
CREATE INDEX IF NOT EXISTS idx_chat_history_created_at 
ON chat_history(created_at DESC);
```

#### Row Level Security (RLS) 設定
- RLSを有効化
- 挿入ポリシー: 全ユーザーが挿入可能（匿名ユーザー含む）
- 読み取りポリシー: 無効化（管理者のみダッシュボードで閲覧可能）
- 更新・削除ポリシー: 無効化
